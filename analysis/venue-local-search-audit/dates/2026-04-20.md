## Observations

- Endpoint behavior is strongly term-sensitive for venue intent:
  - `search=local` returned 3 providers (all `Locales`) in both endpoints.
  - `search=venue`, `place`, `lugar`, `salon`, `espacio para eventos`, `local para eventos` returned 0 in this snapshot.
- `/filtered/full` returned richer address strings but category was missing for the `local` hits; `/filtered` retained the `Locales` category labels.
- Existing runtime behavior depended on exact category term handling; venue-like phrases could stay too literal and miss the only successful keyword (`local`).
- `searchProvidersByCategoryLocation` previously used a single composed query and did not retry aliases or a category-only fallback.

## Findings Promoted

- Promoted root-cause and fix design into `findings.md`:
  - broaden venue alias normalization,
  - retry category-location search with aliases,
  - fallback from `category + location` to `category-only`.
